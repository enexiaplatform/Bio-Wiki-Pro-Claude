# HPLC System Suitability

System suitability testing (SST) confirms that an HPLC system — column, mobile
phase, detector, and software — is performing adequately **on the day**, **before**
you trust any sample results. It's a gate at the start of a run, not part of
method validation.

## The core SST parameters

- **Resolution (Rs)** — separation between adjacent (e.g. analyte and nearest
  impurity) peaks. Too low = peaks overlap = unreliable quantitation.
- **Tailing factor (T)** — peak symmetry; bad tailing distorts integration.
- **Theoretical plates (N)** — column efficiency.
- **% RSD** of replicate standard injections — injection/system precision.
- **Capacity / retention** checks where specified.

These have acceptance criteria defined in the method (and general guidance in USP
&lt;621&gt; Chromatography).

## SST is a gate, not validation

Method **validation** proves the method is fit for purpose (done once, formally).
**SST** is the routine check that the validated method is running correctly today.
If SST fails, you stop — you do **not** report sample results from a run whose
system suitability failed.

## Common pitfalls

- Reporting results from a sequence where bracketing SST injections drifted.
- Aging columns silently degrading resolution until SST finally fails.
- Treating a single passing SST at the start as covering a very long sequence —
  bracket with periodic check standards.

> Tip: trend your SST parameters (resolution, %RSD) over time — a slow decline
> predicts a column or instrument problem before it fails a batch.
