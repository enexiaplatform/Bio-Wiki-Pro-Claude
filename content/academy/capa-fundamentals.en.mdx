# CAPA Fundamentals

Corrective and Preventive Action (CAPA) is how a quality system learns from
failure. Per ICH Q10, a robust CAPA process is expected across the product
lifecycle.

## Correction vs corrective vs preventive

- **Correction** — the immediate fix (e.g. reject the batch).
- **Corrective action** — eliminates the *cause* of the detected problem.
- **Preventive action** — eliminates the cause of a *potential* problem.

## Root cause first

Jumping to actions before establishing root cause is the most common CAPA
finding. Use a structured tool (5 Whys, Ishikawa) and tie each action to a
verified cause.

## The effectiveness check (the part everyone skips)

Define **quantitative, time-bound** criteria *before* closing the CAPA — e.g.
"zero recurrence across the next 3 batches." Evidence must be objective data,
not "training completed."

## When do you even need a CAPA?

Not every deviation needs a full CAPA. A quick triage:

- **Minor, well-understood, one-off** → correction + documented rationale; no CAPA.
- **Recurring, systemic, or high-risk** → full CAPA with root cause.
- **Critical / patient-impact** → CAPA + risk assessment + possible field action.

Over-raising CAPAs buries the system; under-raising them hides systemic problems.
Tie the threshold to risk and trend, and review borderline calls at the quality
review.

## CAPA record — fields that survive an audit

1. Problem statement (what, where, when, how detected, magnitude)
2. Immediate correction + containment
3. Risk assessment (severity, scope, other batches/products affected)
4. Root-cause method and the **verified** root cause
5. Corrective action(s) — each mapped to a cause, with owner + due date
6. Preventive action(s) — extend the fix to similar processes/products
7. **Effectiveness check** — metric, target, time window, owner
8. Closure approval + effectiveness-verification record

## Worked example

*Event:* OOS for assay on Batch 23. **Correction:** quarantine batch, invalidate
result only after a confirmed lab error. **Root cause (5 Whys):** mobile-phase
prepared with an expired buffer because the prep log had no expiry field.
**Corrective:** redo affected analyses; add expiry field + check to the prep log.
**Preventive:** roll the log change across all HPLC methods in the lab.
**Effectiveness check:** zero prep-related OOS across the next 20 batches over 3
months — verified from the OOS trend before closure. *Note "training" is not the
root cause here — the missing log control is.*

> Practitioner tip: if your corrective action is "retrain the analyst," stop. That
> is almost never the true root cause — it's where investigations go to die. Ask
> what in the *system* let the error happen and be undetected.
