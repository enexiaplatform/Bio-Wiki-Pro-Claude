# Computer System Validation (GAMP 5)

Computer System Validation (CSV) provides documented evidence that a GxP
computerized system does what it is supposed to do, consistently and reliably.
GAMP 5 (Second Edition) frames this as a **risk- and science-based** activity,
not a paperwork exercise.

## GAMP software categories

- **Category 1** — infrastructure software (operating systems, databases).
- **Category 3** — non-configured products used as supplied.
- **Category 4** — configured products (e.g. a configured LIMS).
- **Category 5** — custom / bespoke applications (highest risk, most rigor).

## The V-model

User requirements map to acceptance testing; functional specs map to functional
testing; design specs map to installation/operational qualification. Each left
arm has a corresponding right-arm verification.

## Data integrity drives scope

Validation effort should focus where data integrity risk is highest — audit
trails, access controls, electronic signatures, and ALCOA+ attributes. Leverage
supplier documentation for low-risk categories instead of duplicating it.
